Chess Notation Parser - Development Progress

Project Overview

Chess Notation Parser is a small open-source library written in Go that reads chess moves in Standard Algebraic Notation (SAN) and turns them into a structured, machine-readable representation. The goal is to learn parsing techniques and the formal rules behind chess notation by building a tokenizer for moves like Bxe4, e4 or Nf3 from scratch — without external dependencies.

How It Works

A move string is broken down from right to left. The last two characters are always the destination square (file + rank), and the remaining prefix describes the piece, optional disambiguation, and capture:

  • Destination square: the final file (a–h) and rank (1–8) are validated and converted into a 0-based coordinate (a=0 .. h=7, 1=0 .. 8=7).
  • Capture detection: a trailing x in the prefix flags the move as a capture.
  • Piece resolution: the leading character is matched against K, Q, R, B, N — anything else defaults to a pawn.
  • Origin file: for pawn captures like exd5, the source file is extracted from the prefix.

Core Data Model

Every parsed move is reduced to a typed structure that is easy to consume from other programs:

  • Piece: a single-byte enum (P, N, B, R, Q, K).
  • Square: a file/rank pair, using -1 to mark values that are not specified.
  • Move: bundles the piece, origin square, destination square and a capture flag.

The Move type also rebuilds its own notation through a toString() method, which makes round-trip testing straightforward.

Current Progress

The parser already handles the most common cases of algebraic notation:

  • Basic pawn moves and pawn captures.
  • Piece moves with and without captures.
  • File-based disambiguation and input validation with descriptive errors.

Next Steps

Planned improvements to reach full SAN coverage:

  • Castling (O-O and O-O-O).
  • Pawn promotion (e.g. e8=Q).
  • Check and checkmate suffixes (+ and #).
  • Full rank/file disambiguation and a complete test suite.
